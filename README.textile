h1. PLint - Play Lagoon Interface Connector

h2. Description

Plint is a connector module to the LagoonSecurity (LS) application, which is an authentication/autorization system developed by XLM. 
It allows you to easily manage your applciations authentication and autorization.

h2. Adding Module to your project

After downloading the module, unzip it and put it in the same folder as your application, then add to <b>dependencies.yml</b> of your application:

<pre>
require:
    - plint -> plint

repositories:
    - MyModules:
       type:    local
       artifact: "${application.path}/../../../[module]"
       contains:
            - plint
</pre>

And then execute: 
  @$ play deps@
Don't forget to execute @$ play eclipsify@ to enable it in eclipse.

h2. Configuration

After install Plint, in order to use it in your Play! Application, the following configurations must be made on <b>application.conf</b>:

<pre>
lint.protocol=http // Comunication Protocol with LS
lint.port=8080 // Comunication Port with LS
lint.baseUrl=subdomain.lagoonsecurity.dyndns.org // Provided LS Base access URL
lint.login=loginLagoon // Provided LS Login
lint.password=passwordLagoon // Provided LS password
</pre>

The baseUrl, login and password will be provided after registration.

h2. Example Application

In samples-and-tests(https://github.com/dpeixoto/PLint/tree/master/samples-and-tests) you can find an example application using Plint named xcoisas,it simulates an application where users can buy and sell several items.
The URL to enter test application is: @http:\\xcoisas.lvh.me:9000@

Go to LS so you can change the access configurations of the users from xcoisas to the several resources.
